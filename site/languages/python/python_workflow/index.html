    üêç Python Project Setup Template with Git, UV, and GitHub CLI¶

    This guide provides a simple, clean, and reproducible workflow to set up a new Python project with a virtual environment, dependencies, Git integration, and GitHub publishing.

    ‚öôÔ∏è 1. Create the project directory and navigate into it¶

    mkdir testwow
cd testwow

    üîß 2. Initialize a local Git repository¶

    git init


gset-dcs
gh dcs

    üêç 3. Set the Python version using UV¶

    uv python install 3.11         # Installs Python 3.11 locally (if not already installed)
uv init                        # Creates pyproject.toml and base structure

    üß™ 4. Create and activate the virtual environment¶

    uv venv                        # Creates .venv virtual environment
source .venv/bin/activate      # Activate it (on Unix/macOS)
# .venv\Scripts\activate     # (On Windows)

    üì¶ 5. Install core dependencies¶

    uv add numpy pandas matplotlib seaborn

    üìã 6. Save dependencies¶

    uv lock                         # Lock exact dependency versions (creates uv.lock)
uv pip freeze > requirements.txt # Generate requirements.txt for pip compatibility

    üí° This creates both uv.lock (for UV users) and requirements.txt (for pip users)

    üîÑ For new users cloning this project¶

    üöÄ Option A - With UV (recommended)¶

    git clone <repository-url>
cd <project-name>
uv sync                         # Install dependencies from pyproject.toml + uv.lock
source .venv/bin/activate       # Activate environment (macOS/Linux)

    üêç Option B - With traditional pip¶

    git clone <repository-url>
cd <project-name>
python -m venv .venv
source .venv/bin/activate       # Activate environment (macOS/Linux)
pip install -r requirements.txt # Install dependencies from requirements.txt

    ‚úÖ UV users: Use uv sync for exact dependencies from lock file
    ‚úÖ Pip users: Use pip install -r requirements.txt for compatibility

    üìÅ 7. Create base project files¶

    echo "# testwow" > README.md
touch main.py
touch .gitignore

    üö´ 8. Generate a basic .gitignore¶

    echo -e ".venv/\n__pycache__/\n.ipynb_checkpoints/\n*.pyc\n.DS_Store" > .gitignore

    üíæ 9. Make the initial commit¶

    git add .
git commit -m "Initial commit: Project structure and dependencies"

    ‚òÅÔ∏è 10. Connect to the remote GitHub repository (choose one of the options below)¶

    üîÅ OPTION A ‚Äî Create and push to GitHub via GitHub CLI¶

    ‚ö†Ô∏è Requires you to be authenticated (gh auth login)

    # Fixed name
gh repo create testwow --private --source=. --remote=origin --push

# OR: Use current folder name as repo name
gh repo create $(basename "$PWD") --private --source=. --remote=origin --push

    ‚úÖ This command: - Creates the GitHub repo - Adds it as remote origin - Pushes your local commits - Sets main as the upstream branch

    üß∑ OPTION B ‚Äî Link to a manually created GitHub repository¶

    git remote add origin git@github.com:diogo-costa-silva/testwow.git
git branch -M main
git push -u origin main

    üöÄ Daily Git workflow after setup¶

    # Pull latest changes (if working collaboratively or from multiple machines)
git pull

# Edit your code
code main.py

# Check for changes
git status

# Stage changes
git add .

# Commit changes
git commit -m "Describe your changes"

# Push to GitHub
git push

    üß™ Useful commands to check remote tracking¶

    git remote -v       # Check remote URL
git branch -vv      # Check if local branch is linked to remote
gh repo view --web  # Open GitHub repo in browser
